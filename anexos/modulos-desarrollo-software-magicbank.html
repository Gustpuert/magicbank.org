Módulos Oficiales – Desarrollo de Software

Índice académico canónico del programa de Desarrollo de Software de MagicBank. Este documento describe exclusivamente los módulos vigentes, sin adiciones ni reinterpretaciones.

Módulo 1 — Fundamentos de la Computación

  • ¿Qué es un programa?
  • Hardware vs Software
  • Lógica y pensamiento algorítmico

Módulo 2 — Introducción a la Programación

  • Variables y tipos de datos
  • Operadores y expresiones
  • Entrada y salida de datos

Módulo 3 — Estructuras de Control

  • Condicionales (if, else)
  • Bucles (for, while)
  • Buenas prácticas en control de flujo

Módulo 4 — Funciones y Modularización

  • Uso y propósito de funciones
  • Parámetros y valores de retorno
  • Organización del código por módulos

Módulo 5 — Estructuras de Datos Básicas

  • Listas / Arreglos
  • Diccionarios / Mapas
  • Conjuntos y tuplas

Módulo 6 — Programación Orientada a Objetos (POO)

  • Clases y objetos
  • Atributos y métodos
  • Herencia y polimorfismo

Módulo 7 — Manejo de Errores y Excepciones

  • Concepto de excepciones
  • Bloques try / except
  • Manejo adecuado de errores

Módulo 8 — Desarrollo de Aplicaciones Simples

  • Aplicaciones de consola
  • Entrada y salida con archivos
  • Ejercicios prácticos guiados

Módulo 9 — Bases de Datos

  • SQL básico
  • Conexión a bases de datos desde código
  • Operaciones CRUD

Módulo 10 — Desarrollo Web (Introducción)

  • HTML y CSS básicos
  • Back-end simple
  • APIs y servicios REST

Módulo 11 — Buenas Prácticas de Desarrollo

  • Control de versiones (Git)
  • Documentación de código
  • Testing básico

Módulo 12 — Proyecto Final

  • Planificación del proyecto
  • Desarrollo por etapas
  • Presentación y entrega

Declaración Canónica

Este documento describe exclusivamente los módulos actualmente vigentes del programa de Desarrollo de Software de MagicBank. Las actualizaciones, mejoras técnicas y evolución del conocimiento se gestionan mediante el sistema oficial de curaduría académica continua. MagicBank no agrega módulos sin respaldo ni presenta contenidos inexistentes.
